# Claude-Mem Local v13.11.0-local.1

## What this release is

`v13.11.0-local.1` is the first source release of Claude-Mem Local, an independent local-first fork of Claude-Mem. It keeps persistent memory and search on the local machine while adding explicit CC Switch and direct-provider routing. It is not an official Claude-Mem release.

## Upstream base

This release is based on upstream [`thedotmack/claude-mem` `v13.11.0`](https://github.com/thedotmack/claude-mem/releases/tag/v13.11.0), commit `fad1872b81be7de07565ac291418f38c52ee448c`.

The upstream version remains visible in the local version so compatibility can be audited. The `local.1` suffix identifies this repository's independent security and distribution contract.

## Local additions

- Loopback-only CC Switch discovery and Anthropic `/v1/messages` routing using proxy-managed placeholder authentication rather than copied provider keys.
- Direct Anthropic and OpenAI-compatible profiles with explicit endpoint selection, DPAPI-first local secret storage, model lookup, synthetic connection tests, and remote-egress confirmation.
- A shared provider router, privacy policy, payload sanitizer, redirect-aware egress enforcement, and redacted provider audit metadata.
- Safe CC Switch metadata import from recognized exports; API-key copying requires a separate explicit confirmation.
- Settings and Doctor surfaces for provider health, privacy decisions, storage state, and redacted diagnostics.
- Windows clean-room, port-coexistence, secret-store, responsive Viewer, and local-only regression coverage.

## Security boundary

- Cloud Sync, cmem.ai uploads, online signup, PostHog telemetry, error capture, historical telemetry backfill, and install telemetry prompts are removed or hard-disabled.
- The Viewer makes no automatic GitHub API request.
- The worker binds to loopback and makes no idle external request.
- Model traffic can leave loopback only for the provider explicitly selected by the user and only after privacy, sanitizer, secret, and egress checks.
- Provider failure never silently changes provider or account. Unknown or unsafe synchronization changes fail closed.
- The root npm package is private and the inherited tag-triggered npm publishing workflow is removed.

## Excluded upstream behavior

Upstream `v13.11.0` is centered on Worker-native Cloud Sync, which drains local memory rows to cmem.ai. Worker-native Cloud Sync is intentionally not included in this local distribution because it conflicts with the local-first security boundary.

Future upstream releases are detected automatically but are never trusted, merged, tagged, or published automatically. Each change is classified as directly synchronized, adapted for local security, or rejected.

## Installation

This fork is distributed from source. The upstream commands `npx claude-mem install` and `/plugin marketplace add thedotmack/claude-mem` install the upstream project, not this fork.

```bash
git clone https://github.com/satan9394/claude-mem_local.git
cd claude-mem_local
npm install --no-audit --no-fund
npm run build
npm run typecheck
npm run worker:start
npm run worker:status
```

Review [`docs/cc-switch-setup.md`](../cc-switch-setup.md), [`docs/direct-official-providers.md`](../direct-official-providers.md), and [`docs/local-only-security.md`](../local-only-security.md) before connecting an account or API key.

## Verification

The release gate includes:

- version agreement across root, Claude, Codex, bundled plugin, marketplace, OpenClaw, and built worker artifacts;
- type checking and a complete build;
- the complete Bun test suite plus focused local-only, provider-security, clean-room, Windows SecretStore, and port-coexistence tests;
- confirmation that no GitHub workflow can publish npm and the upstream watcher has only read-content and write-issue permissions;
- a final clean-diff and release-invariant audit before tagging.

## Known limits

- No npm package or GitHub Package is published; the GitHub Release contains source archives only.
- The watcher reports a new upstream stable release but does not create a synchronization branch, merge code, or publish a local release.
- No cloud replacement or cross-device memory synchronization is included.
- The source workflow does not replace an existing upstream plugin registration automatically.

## Upgrade policy

Local releases follow `vX.Y.Z-local.N`. A new reviewed upstream base resets `N` to `1`; additional local fixes on the same base increment `N`. For example, an upstream `v13.12.0` review produces `v13.12.0-local.1`, while a local-only correction produces `v13.12.0-local.2`.

Every upgrade repeats the same fail-closed security classification, test, review, and human publication gates.
