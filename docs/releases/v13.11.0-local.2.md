# Claude-Mem Local v13.11.0-local.2

## What this release is

`v13.11.0-local.2` is a security and transparency patch for the independent Claude-Mem Local fork. It makes the difference from upstream visible in the README first viewport and closes a legacy-provider sanitizer gap.

## Upstream base

The reviewed upstream base remains [`thedotmack/claude-mem` `v13.11.0`](https://github.com/thedotmack/claude-mem/releases/tag/v13.11.0), commit `fad1872b81be7de07565ac291418f38c52ee448c`. The `local.2` suffix identifies the second local release on that base.

## Local additions

- Every legacy Claude SDK init/continuation, observation, and summary prompt is sanitized immediately before model egress.
- Privacy diagnostics classify configured legacy loopback gateways as `legacy-loopback-proxy` with `opaque-upstream` visibility and stable warning code `LEGACY_PROXY_UPSTREAM_OPAQUE`.
- Doctor warns when the legacy Claude path uses a configured gateway and recommends explicit CC Switch auto routing.
- The README first viewport now distinguishes local memory from model traffic in Chinese and English and links to a complete destination map.

## Security boundary

- SQLite memory, observations, summaries, prompts, search indexes, logs, and bounded provider audit metadata remain local.
- Model prompts may leave the machine through the explicitly selected provider after sanitization.
- CC Switch at `127.0.0.1` is a transport hop; its selected model provider can be the final remote destination.
- Cloud Sync, cmem.ai uploads, PostHog telemetry/error capture/backfill, online signup, and automatic Viewer GitHub requests remain hard-disabled.
- CC Switch or Direct failure never silently changes provider or account.

## Excluded upstream behavior

Worker-native Cloud Sync and cmem.ai upload behavior remain intentionally not included. Upstream changes are advisory until reviewed and adapted to the local security contract.

## Installation

This fork is distributed from source and GitHub Release archives, not under the upstream npm identity.

```bash
git clone https://github.com/satan9394/claude-mem_local.git
cd claude-mem_local
npm install --no-audit --no-fund
npm run build
npm run typecheck
```

## Verification

The release gate includes focused sanitizer and diagnostic TDD, complete type checking, build/version agreement, the full Bun test suite, live Worker health, CC Switch discovery/activation, a synthetic model request, and confirmation of a new metadata-only CC Switch token-usage row.

## Known limits

- Sanitization reduces accidental credential disclosure but cannot determine whether arbitrary source text is confidential.
- Claude-Mem cannot independently see the upstream destination chosen inside CC Switch; it reports that boundary as `opaque-upstream`.
- The loopback Worker relies on the operating-system user boundary and is not an isolation boundary against a compromised local administrator.
- No cloud replacement or cross-device memory sync is included.

## Upgrade policy

Local releases follow `vX.Y.Z-local.N`. Upstream synchronization is detected but never automatically trusted or merged. Every candidate change receives a security-gated upstream review; unsafe cloud upload, telemetry, silent fallback, or fail-open behavior is rejected or adapted before publication.
